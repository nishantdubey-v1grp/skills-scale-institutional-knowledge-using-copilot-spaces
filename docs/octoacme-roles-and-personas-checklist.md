# OctoAcme Roles & Personas Onboarding & Clarity Checklist

## Purpose
Provide a practical checklist for teams to ensure all key roles are defined, understood, and aligned on responsibilities and interactions.

---

## Project Setup Checklist

When initiating a new project, ensure the following role clarity activities are completed:

### Team Composition
- [ ] Project Manager assigned and available
- [ ] Product Manager assigned and available
- [ ] Development team size and lead identified
- [ ] QA / Testing lead identified
- [ ] Release Manager identified (or designated PM responsibility)
- [ ] Security Champion assigned from development team
- [ ] Data Analyst / Analytics Owner identified
- [ ] UX Researcher / Designer assigned (if applicable)
- [ ] Support Lead engaged (if applicable)
- [ ] Business Analyst engaged (if applicable)
- [ ] Legal / Compliance contact identified (if applicable)
- [ ] Platform / DevOps representative engaged

### Role Clarity & Alignment
- [ ] RACI matrix completed for key decisions and deliverables
- [ ] Escalation paths documented (who escalates to whom and when)
- [ ] Decision ownership defined (who makes final call on different types of decisions)
- [ ] Communication cadence agreed (standups, syncs, reviews)
- [ ] Artifacts and templates shared with the team
- [ ] Roles and personas doc reviewed and acknowledged by all key participants

### Kick-off Alignment
- [ ] Kickoff meeting held with all key roles
- [ ] Success metrics and goals reviewed together
- [ ] Dependencies and cross-team interactions documented
- [ ] Communication preferences and channels established
- [ ] Q&A on responsibilities and interactions addressed

---

## Ongoing Role Health Checks

Regularly review role clarity and interactions:

### Weekly Standups
- [ ] Blockers and dependencies are escalated appropriately
- [ ] Cross-team communication is happening (no siloed decisions)
- [ ] Roles are respecting RACI and decision ownership

### Sprint Planning & Refinement
- [ ] Acceptance criteria are clear to implementers
- [ ] Success metrics are aligned between Product Manager and Data Analyst
- [ ] Risk and compliance considerations are surfaced by Security Champion and Legal/Compliance
- [ ] UX and design guidance is clear from UX Researcher

### Mid-Sprint Check-in
- [ ] Blockers are being escalated to Project Manager appropriately
- [ ] Quality gates are being enforced (code review, security scans, testing)
- [ ] Data instrumentation is on track (Data Analyst)
- [ ] Security guidance is being followed (Security Champion)

### Release Planning
- [ ] Release Manager coordinates all pre-release activities
- [ ] Platform/DevOps environment readiness is verified
- [ ] Support Lead is prepared for post-release monitoring
- [ ] Legal/Compliance sign-off obtained (if required)

### Post-Release
- [ ] Support Lead is triaging customer issues
- [ ] Data Analyst is measuring impact
- [ ] Product Manager is analyzing results and customer feedback
- [ ] Issues are escalated and triaged appropriately

### Retrospective
- [ ] Team reflects on collaboration and communication
- [ ] Role clarity issues or confusion are surfaced
- [ ] Action items to improve cross-functional coordination are created

---

## Collaboration Health Indicators

### Green Flags ✓
- Clear decision ownership and timely decisions
- Cross-team issues are escalated and resolved quickly
- Communication is transparent and happens at the right times
- Roles understand their dependencies and proactively coordinate
- Documentation is kept updated and accessible
- Retrospectives identify and address collaboration gaps
- Support and Quality are engaged early, not just at the end
- Security and compliance considerations are baked into planning

### Red Flags ⚠️
- Decisions are delayed or unclear who owns them
- Work is redone because of unclear acceptance criteria or handoffs
- Cross-team communication is reactive or siloed
- Roles are overlapping or unclear about boundaries
- Documentation is out of date or hard to find
- Security or compliance concerns are raised late
- Support is overwhelmed because issues weren't caught earlier
- Meetings are inefficient or missing key stakeholders

## Role Handoff Checklist

When a team member takes on a new role or leaves, use this checklist:

### New Role Member
- [ ] Review OctoAcme Roles & Personas documentation
- [ ] Review OctoAcme Roles Interaction Matrix
- [ ] Meet with previous role holder (if applicable) to understand context
- [ ] Meet with dependent roles to understand expectations
- [ ] Review current project artifacts (charter, roadmap, risk register, etc.)
- [ ] Observe a few key ceremonies (standups, planning, reviews)
- [ ] Ask questions and clarify expectations
- [ ] Confirm understanding with Project Manager

### Role Transition
- [ ] Document what's in progress and handoff items
- [ ] Brief new role member on current priorities and blockers
- [ ] Introduce new role member to dependent roles
- [ ] Confirm continuity of responsibilities and communication
- [ ] Plan for overlap period if needed

## Quick Role Reference Cards

Use these one-liners when someone is unsure who to talk to:

### "I need to prioritize what to build"
→ **Talk to:** Product Manager + Business Analyst

### "I found a security issue"
→ **Talk to:** Security Champion (then escalate to Security team if critical)

### "We need to measure if this worked"
→ **Talk to:** Data Analyst + Product Manager

### "I need to release this to production"
→ **Talk to:** Release Manager + Platform/DevOps

### "A customer has a problem"
→ **Talk to:** Support Lead (who will escalate as needed)

### "This requires stakeholder approval"
→ **Talk to:** Project Manager + Legal/Compliance (if applicable)

### "I'm blocked and need help"
→ **Talk to:** Project Manager (who will coordinate)

### "I need to understand what users need"
→ **Talk to:** Product Manager + UX Researcher

### "I need infrastructure/DevOps help"
→ **Talk to:** Platform/DevOps Engineer

### "We need to document this for compliance"
→ **Talk to:** Legal/Compliance Liaison + Security Champion
